Design notes

Landing page

Background

The current Civil Service job and career proposition is divided between a job search site and a career site. Both are independently maintained and have different styles: the job search site follows GDS patterns, while the careers site is Wordpress-based and follows a non-GDS design.


Aim

Our aim was to investigate the following:

  • is two separate sites necessary, or could one site cover careers and a job search?
  • would a new site have to be designed outside of GDS constraints (i.e. more 'corporate', more freedom to use imagery, video etc) or would a GDS version be sufficient to engage users?
  • what do users need to know / want to see on this site?
  • would users be happy to link out to existing content (for example on the existing careers site) or would the jump be too distracting for them?
  • is the job search the most important aspect and so have to be prominent on the page?
  • what mediums (e.g. video, PDF, podcast) would be useful to a user?

What we did

We designed two basic landing pages to show to users:

  • a GDS-friendly version (version A), only slightly off GDS standard: large image, uses of video etc
  • an off-standards version (version B), with more imagery, sliding job search and non-standard sections / content display

Version B was designed with the most relevant content on it (at least as far as we could guess, based on previous evidence / desktop research). Version A had some matching content, but was mainly to be used as a simple comparison with B: if A had the same content as B, but displayed in a 'less appealing' way, would it be enough to engage users used to corporate-level job and career sites?


User research

Mural board with testing notes


What we found

  • videos were not well received, both because of the uninspiring static image and because few users we actually interested in watching them
  • it was difficult to get a definitive list of what users would want to see on a candidate landing page, as each user was in their own position (e.g. looking for career info, wanting to search for a CS job, wondering about transferring across CS etc) and thus had their own needs. However, the 'What we offer' and 'How to join' sections were well received generally
  • a 'career finder' tool was considered useful
  • showing some edge case info - like prison leavers, veterans - led to questions about what info we weren't showing. Users felt it was better to cover everyone or not at all, for example by using FAQs to cover all situations
  • all users asked would be happy to answer some short questions that would lead to content targeted to them e.g. don't show info that the user has said is not relevant to them
  • no user struggled with the idea of clicking links which took them to the career site, and some users commented on how they liked the layout of the career site
  • most users preferred version B, but it was felt that this was as much to do with the better content than it was the difference in design from version A
  • some users found version B led to a lot of scrolling, especially due to the large main image
  • most users did not want to read lengthy PDFs or listen to podcasts. However, they did want to see a summary of the info they contained and have the option to go to the PDF or podcast if they wanted to at another time
  • the idea of a single site - covering job search and career info - had a mixed response: some users thought it was a sensible idea, others felt there was no need to muddy the waters if you only wanted to do one or the other
  • the job search was considered very important, and although some users were happy with version B's hidden search feature, most felt it would be best on the page near the top
